# 阶段十四真实输入法手动验证清单

状态：自动化桌面控制通道不可用，以下项目需要在实际 Windows 桌面执行。

## 环境

- Windows 10 或 Windows 11
- 微软拼音
- 用户常用五笔输入法
- 用户常用双拼方案
- 一篇含中文、英文、标点、换行的文章

## 验证步骤

| 场景 | 操作 | 预期结果 |
|---|---|---|
| 拼音连续上屏 | 连续输入至少 100 个中文字符 | 每次上屏只推进对应字符，组合按键不直接推进正文 |
| 五笔连续上屏 | 连续输入至少 100 个中文字符 | 编码按键计入键速，上屏字符参与正文比对 |
| 双拼连续上屏 | 连续输入至少 100 个中文字符 | 组合过程不产生重复字符，候选上屏后推进正文 |
| 组合内退格 | 候选未上屏时按退格 | 正文位置不回退，正式退格次数不增加 |
| 已上屏退格 | 字符上屏后按退格 | 正文位置回退，正式退格次数增加 |
| 换行 | 输入到行尾后按 Enter | 仅目标位置为换行时推进到下一行 |
| 暂停 | 输入若干字符后执行暂停快捷键并等待一分钟 | 暂停期间输入被忽略，恢复后统计耗时不包含暂停时间 |
| 重来 | 执行重来快捷键并确认 | 当前进度和实时统计清空，文章保持不变 |
| 布局切换 | 练习中反复执行布局快捷键 | 进度、错误状态和统计保持不变 |
| 长文章 | 导入接近 50000 字符的文章并连续输入 | 页面可滚动，输入与统计保持响应 |

## 记录要求

发现差异时记录输入法名称、版本、按键顺序、收到的消息阶段（按键或上屏）
以及是否能够稳定复现。IME 消息序列因输入法实现不同，不能仅凭单一输入法结果
断言所有输入法均已完善。
